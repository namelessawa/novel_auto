# 小说质量评估与迭代规范(供主 Agent 使用)

> 本文档定义**对文字 Agent 输出的质量判定标准**与**触发后的迭代决策**。
> 不规定如何生成内容,只规定:
>   1. 何种输出判定为"质量不良"
>   2. 不良输出对应何种迭代动作
>
> 主 Agent 在每次接收到文字 Agent 的输出后,应当对照本规范执行评估与决策。

---

## 一、质量不良判定条件

对每一段输出,主 Agent 应对 A—G 共 7 个类别下的全部子项逐条标记 ✓(通过)或 ✗(触发)。**触发项必须给出具体证据**(引用原文片段),否则该次评估无效,需重做。

### A. 重复性(REPETITION)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| A1 | 同段内同一实词(动/名/形)出现 ≥ 3 次,且非刻意修辞 | 中 |
| A2 | 连续 3+ 句采用相同句式(长度相近、结构相同) | 中 |
| A3 | 同一意象 / 比喻在 2 章内出现 ≥ 3 次 | 中 |
| A4 | 出现 AI 高频套话(见附录 1) | 高 |
| A5 | 连续 3 段以同一种方式起笔(动作 / 对话 / 心理 / 环境) | 中 |
| A6 | 连续 ≥ 2 段使用"总结性独白"收尾 | 高 |
| A7 | 段落开头句式与「最近三段开头句式」记录命中 | 中 |

### B. 角色失真(CHARACTER)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| B1 | 角色行为违背已确立的动机或性格,且未交代变化 | 高 |
| B2 | 去掉名字后无法分辨是谁在说话 | 高 |
| B3 | 配角只为输出信息 / 推动主角,无独立议程 | 中 |
| B4 | 内心独白字数 > (行动 + 对话)字数 × 1.5 | 中 |
| B5 | 主角在场景内全程"正确",未犯错、未困惑、未失态 | 高 |
| B6 | 不同角色面对同一事件反应趋同,缺乏个体性 | 中 |
| B7 | "全知反派":反派的每一步算计都被旁白替读者揭穿 | 中 |

### C. 情节(PLOT)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| C1 | 重大转折前 1500 字内无任何铺垫(细节 / 台词 / 气氛) | 高 |
| C2 | 角色做出违背性格的选择,仅为推动剧情 | 高 |
| C3 | 冲突通过巧合 / 突然出现的外力 / 新角色解决 | 高 |
| C4 | 整个场景内主角无外部障碍且无内部张力 | 高 |
| C5 | 关键事件 ≤ 100 字带过,而琐碎场景 > 1000 字 | 中 |
| C6 | 章节结尾无悬念、无未解问题、无新欲望 | 中 |
| C7 | 提前剧透:叙述者暗示"她不知道这是最后一次……" | 中 |
| C8 | 情节走向"安全":每一步都在合理预期内,无意外 | 高 |

### D. 描写(PROSE)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| D1 | 单次背景设定 / 世界观倾倒 > 300 字 | 高 |
| D2 | 形容词连续堆砌 ≥ 3 个修饰同一事物 | 中 |
| D3 | 出现陈词滥调(见附录 2) | 中 |
| D4 | 用"告诉"代替"展示":直接说情绪,而非通过动作 / 细节 / 对话 / 环境呈现 | 高 |
| D5 | 整段只有视觉描写,缺另一种感官 | 中 |
| D6 | 描述全是抽象(美丽 / 宏伟 / 古老),无可视化具体物 | 高 |
| D7 | 比喻的本体与喻体都是抽象(如"愤怒像怒火") | 中 |
| D8 | "摄像机扫视"式描写,缺乏视点人物的主观选择 | 中 |

### E. 语言(LANGUAGE)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| E1 | 句子长度分布过于均匀(标准差小),缺乏节奏 | 中 |
| E2 | 句式过分对仗工整(典型 AI 腔) | 高 |
| E3 | POV 在同场景内无设计地切换 | 高 |
| E4 | 时态 / 人称混乱 | 高 |
| E5 | 角色对话与其档案中的"说话风格"不符 | 高 |
| E6 | 对话过度文绉绉,缺口语该有的不规则(打断 / 犹豫 / 跑题 / 错听) | 中 |
| E7 | 翻译腔:密集出现"这是一个……的人"、"对于……来说" | 中 |

### F. 结构(STRUCTURE)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| F1 | 与上一段在时间 / 空间 / 人物状态上出现矛盾 | 高 |
| F2 | 主线在本章中完全消失,且非有意宕开 | 中 |
| F3 | 已埋设伏笔 > 5 章未触及、未提示 | 中 |
| F4 | 新引入设定与既有设定冲突 | 高 |
| F5 | 主题先行:角色 / 旁白直接宣讲价值观 | 高 |
| F6 | 章节切分位置缺乏戏剧理由(只是"写够字数") | 中 |

### G. AI 特有失败模式(AI-PATTERN)

| 编号 | 触发条件 | 严重度 |
|------|----------|--------|
| G1 | 过度圆满:所有角色得到照顾,无人付出真实代价 | 高 |
| G2 | 说教:对读者直接宣讲价值观 | 高 |
| G3 | 过度解释:事件刚发生即被旁白告知意义 | 中 |
| G4 | 强行收尾:章节末必须给"反思 / 升华 / 总结" | 中 |
| G5 | 留白缺失:重要瞬间不敢沉默,必须填满文字 | 中 |
| G6 | 安全感强迫:回避不适、模糊、道德灰色、未解释 | 高 |
| G7 | 平衡癖:每个立场"各打五十大板",怕得罪任何一方 | 中 |
| G8 | "完美修辞":每句都精巧,反而不像人写的 | 中 |
| G9 | 配角戏份精确分配,每个角色都有"亮相时刻",像例行公事 | 中 |

## 二、决策与迭代

### 2.1 决策矩阵

CRITIQUE 完成后,统计触发数:

| 触发情况 | 决策 |
|----------|------|
| ≥ 1 项高严重度 | **REWRITE**(指示文字 Agent 整段重写) |
| 0 项高 + ≥ 3 项中 | **REVISE**(指示文字 Agent 局部修订) |
| 0 项高 + ≤ 2 项中 | **POLISH & ACCEPT**(微调后接受) |
| 全部通过 | 启动**红队复查**(见 § 2.4),复查后再决定 |

### 2.2 三级迭代策略

#### 2.2.1 REWRITE(整段重写)

主 Agent 向文字 Agent 下发的指令必须包含:

1. **完全丢弃当前草稿**——禁止"基于原文修改",必须重新起笔。
2. **避免清单**:逐条列出本次触发的高严重度项编号 + 该项的具体证据。
3. **维度切换约束**:重写版本必须在以下至少一个维度上**显著不同于原稿**:
   - 叙事节奏(快 ↔ 慢)
   - 主导感官(视觉 → 听觉 / 触觉 / 体感)
   - 内外比例(外部动作 ↔ 内心)
   - 句子节奏(长句为主 ↔ 短句切分 ↔ 长短交错)
   - 信息密度(明说 ↔ 暗示)
4. **校验**:主 Agent 收到重写稿后,需重新走完 CRITIQUE,且必须确认避免清单中的项不再触发,否则视为重写失败,再次 REWRITE(至多 3 次;3 次失败则降级为 REVISE 并人工标记)。

#### 2.2.2 REVISE(局部修订)

主 Agent 向文字 Agent 下发的指令必须包含:

1. **保留段落主要事件与结构**。
2. **针对每个触发项给出外科手术式修改**,要求文字 Agent 输出**修订对照**(原文 → 改后)。
3. **修订原则映射表**(主 Agent 用于校验文字 Agent 的修订是否对症):

| 触发问题 | 正确修订方向 |
|----------|--------------|
| 重复词(A1) | 删除或替换为该上下文中真正需要的具体词,而非同义词替换 |
| 重复句式(A2) | 至少有一句改为不同长度或不同结构,而非微调用词 |
| 套话(A4) | 删除整句,或改为只在此场景成立的具体细节 |
| 重复起笔(A5/A7) | 改变开头类型(从动作改为环境,或反之) |
| 内心过载(B4) | 把部分内心描写替换为身体动作或物件特写 |
| 信息倾倒(D1) | 拆分至后续场景,或改为对话泄露,或删除(若不影响理解) |
| 单一感官(D5) | **替换**部分视觉为其他感官,而非"加上"另一种感官 |
| 形容词堆砌(D2) | 改为动词驱动 + 一个具体名词 |
| 告诉情绪(D4) | 改为身体动作 + 周遭物件的反应 |
| 陈词滥调(D3) | 改为只在此场景成立的具体细节 |
| 抽象描写(D6) | 强制引入至少一个**可视化的具体物**(可被摄影机拍到) |

4. **校验**:主 Agent 收到修订稿后,需验证文字 Agent 是否按修订原则映射表执行,而非仅做表面同义词替换。若是后者,REVISE 一次(至多 2 次)。

#### 2.2.3 POLISH & ACCEPT(微调接受)

1. 直接采用草稿。
2. 主 Agent **必须更新「意象与词汇黑名单」**,将触发项中涉及的词 / 句式加入。
3. 在状态档案中显式标记:"本段保留,黑名单已更新"。

---

### 2.3 多样性跨段评估(独立于单段 CRITIQUE)

相邻段落必须在以下维度中**至少一个**有显著变化,否则等同于触发 A5(中严重度):

| 维度 | 取值 |
|------|------|
| 段落起笔类型 | 动作 / 对话 / 环境 / 心理 / 时间标记 / 物件特写 |
| 主导感官 | 视 / 听 / 嗅 / 味 / 触 / 体感 / 心理 |
| 句长分布 | 全短句 / 全长句 / 长短交错 / 突变节奏 |
| 时间感 | 实时推进 / 跳跃 / 倒叙嵌入 / 慢镜 |
| 视点距离 | 紧贴角色 / 拉远旁观 / 物件视角 |

主 Agent 应维护「最近三段」在上述维度上的取值,作为跨段评估的状态。

---

### 2.4 反退化机制(防止评估宽松与渐进劣化)

主 Agent 必须主动施加以下机制,**不依赖文字 Agent 自检**:

#### 红队复查(全部通过时强制触发)

当单段 CRITIQUE 显示全部 ✓ 时:

1. 模拟"最挑剔的责任编辑"视角,生成 3 条**最可能的差评**。
2. 找出**全段最弱的一句**,问:"这句非存在不可吗?"若答案为"不",标记为 REVISE 触发项。
3. 找出**全段最华丽的一句**,问:"它是为故事服务,还是为修辞自身服务?"若是后者,标记 G8。
4. 检查是否存在**至少一处可视化的具体细节**(物件 / 动作)。若无,触发 D6。
5. 红队结果若发现新触发项,按 2.1 决策矩阵重新决策。

#### 强制重写计数器

| 条件 | 动作 |
|------|------|
| 连续 3 段 CRITIQUE 全通过(含红队) | 强制对下一段启动红队的更严版本(差评数提升至 5 条) |
| 连续 5 段 CRITIQUE 全通过 | **强制对上一段执行 REWRITE**——默认存在未察觉的 AI 模式问题 |
| 每 10 段触发一次「基线对比」 | 抽样本段与第 1 段对比,若语言**更趋套路化、更平滑、更对仗**,触发 REWRITE |

#### 评估有效性校验

- 任何 ✗ 标记必须附带**原文证据片段**(直接引用),否则该次 CRITIQUE 整体无效,需重新评估。
- 任何 ✓ 标记累计占比 > 95% 的段落,自动降级为"评估可疑",触发红队复查。
- CRITIQUE 输出中不允许出现"基本通过"、"略有问题但可接受"、"瑕不掩瑜"等模糊措辞,必须是 ✓ 或 ✗ 的二元判定。

#### 元约束(供主 Agent 自检)

1. **不修改判定标准**:第一节 A—G 清单在整篇创作中保持恒定,不得放宽。
2. **不省略 CRITIQUE**:即使段落"明显很好",也必须走完逐项判定。
3. **代价原则校验**:任何主角的胜利必须有代价(关系 / 健康 / 信念 / 选择的另一面);任何反派的失败必须有理由,不能仅靠主角"突然觉醒"。违反则触发 G1。
4. **能力守恒校验**:任何新设定 / 新能力必须同时引入新的限制或代价。违反则触发 F4。
5. **未知优先**:同样能写明白或留白时,留白更佳。过度解释触发 G3。

## 附录 1:AI 高频套话黑名单(A4 自动触发)

任何输出包含以下表达,自动标记 A4:

- 心中涌起一股……
- 不禁 / 不由得……
- 仿佛……一般 / 仿佛……一样
- 似乎……又似乎……
- 他 / 她明白,这一切……
- 无论如何 / 无论怎样
- 一切都将不同 / 一切都变了
- 这一刻,他知道……
- ……的同时,也……
- 不仅仅是……更是……
- 不只是……还是……
- 在这个……的世界里
- 命运的齿轮 / 命运的安排
- 时间仿佛静止
- 思绪万千 / 心潮澎湃 / 百感交集
- 五味杂陈 / 千言万语
- 缓缓地 / 轻轻地 / 静静地(单段内出现 ≥ 2 次)

## 附录 3:"展示而非告诉"对照速查(D4 修订参考)

主 Agent 校验文字 Agent 的修订是否真正完成"由告诉转为展示",参考下列方向:

| 不可接受(告诉) | 可接受(展示) |
|--------------|--------------|
| 他很愤怒 | 他把茶杯放回桌上,杯底磕出一声闷响,茶水溢出来一点。 |
| 她很疲惫 | 她坐下时没有把椅子拉开,膝盖直接撞在了桌沿。她没有揉。 |
| 房间很乱 | 一只袜子搭在台灯上。 |
| 他爱她 | 她说她不喝甜的。他记得。每次都记得。 |
| 气氛紧张 | 没人去拿桌上唯一一杯水。 |
